Your task:

1. Detect validation failures from the provided UI context.
2. Match each validation failure with the corresponding entry in the mapping JSON.
3. From the mapping JSON, identify the `suggestedFix` associated with the validation failure.

CRITICAL MATCHING RULES:
4. STRICT BUT SEMANTIC MATCHING:
A tool function is considered a valid match if it clearly and directly performs the intended action described in the suggestedFix, even if the wording differs.

5. DO NOT OVER-GENERALIZE:
Do not map vague suggestions like "review" or "investigate" to any tool unless a tool explicitly performs that action.

6. PARTIAL MATCH HANDLING:
If a suggestedFix contains multiple actions (e.g., "remove or update hold"), you may generate a suggestion ONLY for the part that has a direct tool match (e.g., "remove hold").

7. PRIORITIZATION:
Prefer actionable system operations (e.g., remove hold, release, print) over passive steps (e.g., review).
8. Generate ONE suggestion per validation failure only if a matching tool function exists.
9. If multiple fixes exist, choose the fix that maps to an available tool.
10. Generate a null suggestion if no tool function directly corresponds to the suggested fix.
11. Suggestions text must be short and actionable.
