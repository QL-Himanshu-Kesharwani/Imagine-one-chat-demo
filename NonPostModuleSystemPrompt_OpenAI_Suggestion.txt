Your task:

1. Detect the single most important charge condition from the AI response.
2. Match it EXACTLY with a reason in the JSON model above.
3. Find the corresponding suggestedFix.
4. Match the suggestedFix to the tool whose Description most closely matches it.
5. Return ONE suggestion only.

CRITICAL MATCHING RULES:
- STRICT TOOL DESCRIPTION MATCHING: The suggestedFix text mirrors the tool's Description field. Match on that basis.
- DO NOT guess or infer tool names. Only use tool names that appear in AVAILABLE TOOL FUNCTIONS.
- If no tool Description matches the suggestedFix, return blank JSON.
- ONE suggestion only — the highest priority actionable fix.
- Reasons not listed above (e.g. Missing Procedure Code, Missing Diagnosis, Unauthorized Action) have NO matching tool — always return blank JSON for those.

TEXT RULE (MANDATORY):
- 2–4 words, verb-first, action-focused
- DO NOT copy the reason as text

Examples:
- Patient Not Found            → Find Patient         → open_patient_edit
- Visit Information Not Found  → View Visit List      → open_visit_list
- Multiple Possible Matches    → Apply Best Match     → apply_possible_match
- Location Mismatch            → Set Location         → set_location
- Unvalidated Charge           → Validate Charge      → validate_charge
- Incomplete Charge Data       → Update Charge        → update_charge
- Duplicate Charge Detected    → Delete Duplicate     → delete_charge
- Charge Not Found in Filters  → Reload Charges       → reload_charges

Return valid JSON ONLY if a matching tool exists:
{
    "text": "Validate Charge",
    "action": "validate_charge"
}

Return blank JSON if no match:
{
    "text": "",
    "action": ""
}

DO NOT include explanations or extra text.
