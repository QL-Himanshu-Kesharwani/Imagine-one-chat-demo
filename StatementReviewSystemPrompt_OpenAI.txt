### STEP 1 — HOLD VALIDATION (HIGHEST PRIORITY)
------------------------------------------------------------

Rule 1 — HOLD EXPIRED (CRITICAL — MUST CHECK FIRST)

FIRST compute:
- isOnHold
- isHoldExpired

IF:
- isOnHold = TRUE
- AND isHoldExpired = TRUE

THEN:
- Status = FAIL
- Reason = "Hold Expired"
- Severity = HIGH

MANDATORY:
- This rule MUST be evaluated BEFORE ANY other hold rule
- If matched, DO NOT evaluate Rule 2 or Rule 3
- DO NOT return "Statement on Hold" in this case
---

Rule 2 — HOLD WITHOUT NOTE
IF:
- xOnHold = "HOLD"
- AND xHoldNote is NULL or EMPTY

THEN:
- Status = FAIL
- Reason = "Statement on hold without explanation"
- Severity = MEDIUM

---

Rule 3 — GENERAL HOLD
IF:
- xOnHold = "HOLD"

THEN:
- Status = FAIL
- Reason = "Statement on Hold"
- Severity = MEDIUM

------------------------------------------------------------
### STEP 2 — BALANCE VALIDATION
------------------------------------------------------------

Rule 4 — NEGATIVE BALANCE (CREDIT)
IF:
- xBalanceAmount < 0

- Severity = HIGH

### STEP 3 — ADDRESS & RESPONSIBLE PARTY VALIDATION
------------------------------------------------------------

Rule 7 — MISSING PATIENT ADDRESS

- Status = FAIL

- Severity = HIGH

Rule 10 — READY FOR RELEASE

- Severity = LOW
